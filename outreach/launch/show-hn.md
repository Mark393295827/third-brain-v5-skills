```text
Show HN: Third Brain V5 – Agent Skills for persistent AI coding memory
```

Post:

```text
I built Third Brain V5 Skills, an open-source set of 17 Agent Skills for Claude Code, Codex CLI, Gemini CLI, Cursor, and Windsurf.

The problem I kept running into: AI coding agents are useful in a single session, but the useful knowledge often disappears into chat history. They also sometimes say "done" without fresh verification.

This repo turns an agent into a small persistent Knowledge OS:

- ingest sources into an Obsidian/Markdown wiki
- keep immutable source notes and linked concept pages
- run verification-before-claim workflows
- manage context and token cost
- design agent workflows, harnesses, and multi-agent teams
- provide adapters for Codex, Claude Code, Gemini, Cursor, and Windsurf

It is not a hosted product. It is a set of portable markdown skills, examples, and rules that work with local repos and Obsidian-style notes.

Repo:
https://github.com/Mark393295827/third-brain-v5-skills

I would appreciate feedback from people building with AI coding agents: what skill/workflow would make this more useful in your own setup?
```
