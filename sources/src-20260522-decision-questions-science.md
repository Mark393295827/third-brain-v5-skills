# 七个决策问题的行为科学映射

## Source Summary

这条今日感悟把一组个人决策提示词映射到行为经济学、组织学习、敏捷迭代、精益创业、系统思维和创造力研究。核心不是证明某个单点技巧有效，而是把七个问题组合成一个可执行的个人决策操作系统：先让未来自我显像，再区分探索/利用，随后用两周交付、自动化实验、二阶后果、单一焦点和五分钟决策时间盒约束行动。 ^src-decision-questions-summary

## Import Notes

1. “未来的我会感谢什么？”用于增强未来自我连续性，降低时间折扣带来的短视偏差；用户将其理解为 Working Backwards 的心理学基础。 ^ki-future-self

2. “这是 70% 大赌注还是 30% 例行？”把决策显式标记为探索型或利用型动作，借鉴 March 的探索/利用框架和 70-20-10 创新资源分配实践。 ^ki-exploration-budget

3. “一个人能在 2 周内发货吗？”把 Scrum 冲刺和时间盒方法下沉到个人项目，强迫定义最小可交付版本并抑制范围膨胀。 ^ki-two-week-ship

4. “能不能把它做成自动化实验？”把决策转译为假设、指标和自动收数机制，对应 Lean Startup 的 Build-Measure-Learn、MVP 和 validated learning。 ^ki-automated-experiment

5. “如果成功了，会不会搞坏什么？”把注意力从一阶成功转向二阶/三阶后果，要求预演成功本身可能制造的瓶颈、副作用和系统压力。 ^ki-success-disaster

6. “如果只能做一件事，我会选什么？”用极端约束消除选择过载，寻找真正的高杠杆路径，并与 One Metric That Matters 的单指标思想相通。 ^ki-one-thing

7. “5 分钟内决定并进入执行”把决策过程本身时间盒化，用有限时长对抗分析瘫痪、完美主义和认知负荷。 ^ki-five-minute-decision

## Key Insights

1. 这组问题的价值来自组合：它同时覆盖时间视角、风险预算、交付周期、验证机制、系统副作用、焦点选择和行动启动。
2. 它不是单纯的自我提问清单，而是一个轻量决策协议：每个问题都把抽象判断压缩成一个可执行约束。
