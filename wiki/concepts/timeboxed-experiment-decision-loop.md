# 时间盒实验决策循环

> 时间盒实验决策循环把“想清楚”改写为“在固定时间内交付最小实验、收集指标、再学习”，用时间边界和自动化反馈控制范围膨胀。
> (Source: [[src-20260522-decision-questions-science#^ki-two-week-ship]], [[src-20260522-decision-questions-science#^ki-automated-experiment]]) WARNING: Single source

## Core Mechanism

```text
两周边界
  -> 定义最小可交付版本
  -> 自动化收数
  -> 衡量唯一关键指标
  -> 学习后继续/调整/停止
```

用户笔记把两周交付、自动化实验和五分钟决策放在同一链条里：先限制项目周期，再把决策变成实验，最后限制决策时长，迫使系统从分析进入执行。 (Source: [[src-20260522-decision-questions-science#^ki-two-week-ship]], [[src-20260522-decision-questions-science#^ki-automated-experiment]], [[src-20260522-decision-questions-science#^ki-five-minute-decision]]) WARNING: Single source

## Classifications / Comparisons

| Layer | Constraint | Output |
|---|---|---|
| Decision | 5 minutes | enough-to-act conclusion |
| Delivery | 2 weeks | usable increment |
| Learning | automated measurement | validated or invalidated assumption |

## Implications / Applications

- 单人项目可以用“两周内能否发货”作为第一道 scope filter，不能发货的想法需要缩小到可交付版本。 (Source: [[src-20260522-decision-questions-science#^ki-two-week-ship]]) WARNING: Single source
- 自动化实验要求提前写清楚假设、指标和收数机制；否则项目容易退化成无法检验的主观努力。 (Source: [[src-20260522-decision-questions-science#^ki-automated-experiment]]) WARNING: Single source
- 五分钟决策不是否定深度思考，而是为低到中等风险选择设置行动启动阈值。 (Source: [[src-20260522-decision-questions-science#^ki-five-minute-decision]]) WARNING: Single source
