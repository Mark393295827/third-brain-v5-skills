# 成功灾难二阶思维

> 成功灾难二阶思维不是问方案是否会成功，而是问成功以后会在哪里制造新的瓶颈、副作用或系统压力。
> (Source: [[src-20260522-decision-questions-science#^ki-success-disaster]]) WARNING: Single source

## Core Mechanism

```text
假设成功
  -> 识别结果的结果
  -> 查找新瓶颈
  -> 预设缓冲/降级/约束
  -> 降低成功后的系统性损害
```

用户笔记将“如果成功了，会不会搞坏什么？”视为二阶/三阶后果和 premortem 的一句话版本，重点检查成功场景本身引发的下游问题。 (Source: [[src-20260522-decision-questions-science#^ki-success-disaster]]) WARNING: Single source

## Classifications / Comparisons

| Lens | Question | Typical blind spot |
|---|---|---|
| First order | 这会不会有效？ | 有效后的承载能力 |
| Second order | 有效以后会发生什么？ | 新的激励和瓶颈 |
| Success disaster | 如果大获成功，哪里先坏？ | 用户、团队、系统、现金流或信誉压力 |

## Implications / Applications

- 对增长、发布、自动化和团队流程改造，成功灾难检查应在执行前完成，因为这些动作一旦有效，常会放大原有系统薄弱点。 (Source: [[src-20260522-decision-questions-science#^ki-success-disaster]]) WARNING: Single source
- 该检查适合接在 [[timeboxed-experiment-decision-loop|时间盒实验决策循环]] 之前：实验不仅要定义成功指标，也要定义成功后的承载边界。 (Source: [[src-20260522-decision-questions-science#^ki-automated-experiment]], [[src-20260522-decision-questions-science#^ki-success-disaster]]) WARNING: Single source
